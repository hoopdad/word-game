# Agent & Skill Optimization Report
Generated: 2026-06-18

## Analysis Summary

Examined 9 sessions from the last 24 hours across word-game-* repos.
Total tool calls analyzed: 845+ bash, 440+ view, 120+ edit, 12+ sleep commands.

### Key Inefficiencies Identified

1. **Sleep-based deployment waits** — 12+ `sleep N` commands per orchestration session
   instead of polling with timeout. Each sleep wasted 15-180 seconds of wall-clock time.

2. **Repeated `find` commands** — 20+ find/ls calls to locate files whose paths are
   stable and documented (e.g., useAuth.ts, nginx.conf.template, Dockerfile).

3. **Multiple Azure AD queries** — 15+ individual `az ad app show` calls fetching
   one field at a time instead of compound `--query` with multiple fields.

4. **Build-deploy-debug loops** — Web image rebuilt 5+ times because MSAL config
   wasn't validated BEFORE building. Each cycle took ~3 minutes.

5. **Undocumented route flow** — Sessions spent 10+ turns tracing user→WAF→API
   routing because no single reference existed for the request flow.

6. **Agent instructions too verbose** — Critics had 80+ line CI/CD workflow checklists
   that are irrelevant post-migration to local azd deploy. Tokens wasted reading
   outdated requirements.

## Prompting Improvement Suggestions

### For the orchestrator (word-game-harness)

1. **Pre-flight validation before ANY deployment**: Before calling `azd-deploy.sh`,
   always run `scripts/check-msal-config.sh` and verify all container apps are healthy.
   This catches 80% of post-deploy failures before they happen.

2. **Use topology.md as context injection**: When starting troubleshooting sessions,
   prime with: "Consult .copilot/topology.md for all file locations, IDs, and flow."
   This single line saves 10+ turns of discovery per session.

3. **Explicit skill invocation in prompts**: Instead of "fix the auth issue", prompt
   with "use the entra-vite-spa-auth skill to diagnose blank popup." Skills carry
   structured diagnostic protocols that prevent ad-hoc debugging.

4. **Batch diagnosis, then batch fixes**: Prompt pattern should be:
   "Diagnose all issues first (do not fix yet), then present findings, then fix all
   in one pass." This prevents the edit→deploy→discover-next-issue→edit→deploy loop.

### For specialist agents

5. **"Contract-first" prompting**: Always start work requests with "Read the contract
   at .contracts/game-api.yml FIRST, then implement." This prevents field-name and
   route-path misalignment that caused the 405 bug.

6. **Validation as exit gate, not iterative**: Prompt pattern: "Make ALL changes first,
   then run validation ONCE at the end." Not: "Run lint after each file change."

7. **Compound commands over separate turns**: Encode in prompts that agents should
   chain: `ruff check . && ruff format --check . && pytest tests/ && docker build -t x:ci .`
   as one command, not four separate tool calls.

### For troubleshooting sessions

8. **Screenshot + skill pattern**: When user provides a screenshot with an error,
   the ideal prompt structure is: "[skill name] + symptom + deployed URL."
   Example: "entra-vite-spa-auth: blank popup on login. URL: https://word-game-waf..."

9. **Deploy-and-verify as atomic operation**: After every fix+deploy, immediately run
   `scripts/verify-deploy.sh`. Encode this as: "After deploying, run verify-deploy.sh
   and report the results." This catches regressions in the same turn.

10. **Eliminate Azure CLI exploration**: Instead of `az containerapp show ... | jq`,
    use the MCP tools (`diagnose_container_app`, `inspect_container_app`) which return
    structured data. They save 3-5 turns of manual CLI exploration per issue.
